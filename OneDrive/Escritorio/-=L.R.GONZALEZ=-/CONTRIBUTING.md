# CONTRIBUTING

Status: draft for future clean repos.

## Rules

- Work from `AGENTS.md`.
- Keep layers separate: open, commercial, books/editorial, private game, archive.
- Do not touch private game paths unless explicitly assigned.
- Do not delete files. Archive or list candidates.
- Do not claim readiness without tests or explicit checklist evidence.

## Change Flow

1. Read current audit and risk docs.
2. Check working tree.
3. Make focused changes.
4. Run relevant checks.
5. Update docs, manifests, and migration maps.
6. Report residue.

## PR Expectations

Every PR should include:

- summary;
- files changed;
- commands run;
- test results;
- risks;
- private-game exclusion confirmation;
- secret-scan result.
