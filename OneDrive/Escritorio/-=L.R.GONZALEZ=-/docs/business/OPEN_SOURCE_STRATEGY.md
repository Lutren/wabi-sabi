# OPEN_SOURCE_STRATEGY

Status: first public GitHub batch published with evidence on 2026-05-02.

## Goal

Use free developer tools to attract community without giving away paid products, full books, private lore, or game source.

## 2026-05-02 Doctrine: Open Source Max + Patrimony Guard

The operator preference is to open source everything that can be opened without
destroying family patrimony, privacy or security.

Operational update: the selected commercial model is `open core + UI paga`.
Open repos should expose the reusable technology, schemas, CLIs, synthetic
fixtures and low-claim docs. The paid surface is the polished UI, wrapper,
agent workflow, installer, support, hosted operation and premium templates.

Open by default:

- source code for public-safe tools;
- schemas, validators, CLIs and synthetic fixtures;
- reproducible examples, benchmarks and whitepapers;
- generic curador/data-catalog workflows.

Protect by default:

- MEDIOEVO books, full canon, RPG/TCG, assets and story data;
- commercial app binaries, premium templates, paid bundles and support playbooks;
- private prompts, sessions, account data, tokens and local automation state;
- brand, domains and monetization infrastructure.

## What To Give Away

- SDKs.
- CLI tools.
- Templates.
- Small sample datasets.
- Public-safe docs.
- Examples that do not need private assets.

## What Not To Give Away

- Full books.
- Commercial app installers.
- Private game/TCG source.
- Game lore and assets.
- Gumroad paid bundles.
- Secrets, sessions, or vendor dumps.

## License

No default license yet. Mark all candidates `LEGAL_REVIEW_REQUIRED` until each package is reviewed.

## First Recommended Release

The first local batch is already built under `releases\free-dev`. Recommended external order:

1. `observacionismo-gate`, because it is the smallest and cleanest public API.
2. `obsai-core`, because it gives the method a reusable technical core.
3. `residueos`, because it is useful but needs careful `DEMO_ONLY` framing.
4. `gemma-observacionismo-cleanup`, only as a cleanup/observation helper, never as model-weight work.
5. `obs-safe-integration-kit`, as the clearest wrapper/adoption kernel for third-party agent projects.
6. `claudio-os-blueprint`, only as blueprint/handoff, not as an ISO or finished OS.

The free-dev batch now has six local ZIPs, manifests and smoke evidence. Four of those open-dev repos were published to GitHub; `claudio-os-blueprint` and `gemma-observacionismo-cleanup` remain local package candidates until a separate destination review.
